### 3. stop_loss_service.py — Class C — CRITICAL FIX

**Change:** Max-loss-price now uses actual entry price (from position avgPrice) and divides max_loss_pct by leverage.

**After:** `max_loss_price = actual_entry * (1 - max_loss_pct / leverage)` — at 15x with 10% max loss, triggers at 0.67% price move = 10% leveraged loss.

**Change:** ~260 lines of new code. After profitable exits during strong trends, bot watches for a pullback and re-enters. Can bypass entry gate and smart entry safety checks.

**Configuration:** ENABLED=True, ENTRY_GATE_BYPASS=True, SMART_ENTRY_BYPASS=True (ALL LIVE)

| Check | Result |
|-------|--------|
| Root cause proven? | N/A — new feature, not a fix |
| Diff bounded? | YES for the feature itself — self-contained methods |
| Regression risk? | **HIGH** — bypasses entry gate and smart entry, new entry paths |
| Trading behavior changed? | **YES** — fundamental change to entry logic |
| Replay/paper/shadow evidence? | **NONE** |
| Rollback straightforward? | YES — set PULLBACK_REENTRY_ENABLED=False |

**Guard analysis:**
- Cancel conditions: time_expired (30min), adx_collapsed (<20), regime_flipped, pullback_too_deep (>3%), mode_changed, not_running
- Requires: min pullback 0.5%, MR extension eased to moderate, RSI zone, candle/MACD confirmation
- Limits: max 3 re-entries per session, 5min cooldown after cancel
- Mode switching blocked during watch

**Concerns:**
1. Entry gate bypass is active LIVE — this is a Class C trading-path change with zero shadow evidence
2. Smart entry bypass removes RSI/EMA checks that normally prevent bad entries
3. Feature was not released through shadow/advisory pipeline like Phase 7 profit protection was
4. If the trend analysis is wrong (e.g., ADX still high but trend actually reversing), the bot will re-enter into a reversal with safety gates bypassed

**Gate Decision: NEEDS_MORE_VALIDATION**
**Risk Class: C**
The feature code is well-structured with comprehensive guards. However, it fundamentally changes entry behavior by bypassing safety gates, and there is zero replay/paper/shadow evidence. Recommend:
1. Set `PULLBACK_REENTRY_ENABLED = False` until shadow evidence is gathered
2. Run in advisory_only mode first (like Phase 7 profit protection)
3. At minimum, disable the gate bypasses (`ENTRY_GATE_BYPASS=False`, `SMART_ENTRY_BYPASS=False`) and let pullback re-entries go through normal safety checks
