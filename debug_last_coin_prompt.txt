## AI Agent Task: Debug "Last Coin Quick-Run" Feature

### Server Access
- SSH: root@178.18.245.6
- Project Folder: /var/www/opus_trader/

### Goal
Investigate why the "Last Coin Quick-Run" feature's HTML section is not appearing in the browser despite being present in the template file.

### Context
A "Last Coin Quick-Run" feature was added to allow users to quickly restart the last bot they started. The feature consists of:

1. **HTML Section** (added to `templates/dashboard.html` at lines 663-676):
   - A hidden section with id `last-coin-section`
   - Contains elements: `last-coin-symbol`, `last-coin-mode`, `btn-run-last-coin`, `run-last-coin-text`
   - Located right before the "Active Bots Table" section

2. **JavaScript Functions** (added to `static/js/app_lf.js`):
   - `LAST_STARTED_BOT_KEY` constant at line 7
   - `saveLastStartedBot()`, `updateLastCoinDisplay()`, `runLastCoin()` functions after line 224
   - Modified `botAction()` to capture bot data before API call
   - `updateLastCoinDisplay()` called on page load

### The Problem
The JavaScript is working correctly (console shows):
```
[LastCoin] saveLastStartedBot called with: {...}
[LastCoin] Saved to localStorage: {id: '...', symbol: 'RIVERUSDT', mode: 'long'}
[LastCoin] updateLastCoinDisplay - section: null symbolEl: null modeEl: null
[LastCoin] ERROR: section element not found!
```

**The HTML elements return `null` when queried with `document.getElementById()`**, even though:
- The section exists in `templates/dashboard.html` (verified with grep/cat)
- Flask is configured with `TEMPLATES_AUTO_RELOAD = True` and `debug_mode = True`
- The server has been restarted multiple times
- Hard refresh (`Ctrl+Shift+R`) has been performed

### Files to Investigate

1. **`templates/dashboard.html`** - Verify the section exists at lines 663-676
2. **`dashboard.html`** (root folder) - There's a SECOND dashboard.html in the root that may be interfering
3. **`static/js/app_lf.js`** - The active JS file (not `app.js`)
4. **`static/js/app_v5.js`** - Also loaded, may have conflicting code
5. **`app.py`** - Flask configuration at line 75, template rendering at line 422

### Key Questions to Answer

1. **Is Flask serving from the correct template file?**
   - Flask uses `render_template("dashboard.html")` at line 422
   - Template folder is `templates/` (default)
   - But there's also a `dashboard.html` in the project root - is this causing issues?

2. **Is the HTML actually being served to the browser?**
   - Check the actual page source served by the server
   - Search for `last-coin-section` in the served HTML
   - Command: `curl -s http://127.0.0.1:8000/ | grep "last-coin-section"`

3. **Is there a caching issue?**
   - Flask template caching
   - Browser caching
   - Jinja2 bytecode cache
   - Nginx/reverse proxy caching (if applicable)

4. **Is there another script or code removing/hiding the element?**
   - Check if any JavaScript might be removing the element
   - Check if Jinja2 conditionals might be hiding it

5. **Are the JS files synced between local and server?**
   - Verify `static/js/app_lf.js` on server has the Last Coin functions
   - Check version strings match
